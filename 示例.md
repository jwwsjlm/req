# req 新手示例

这份文档把你当成第一次写 Go HTTP 请求的人来写。你不需要一次看完，先从“最小项目”跑通，再按自己要做的事去找对应示例。

本文所有示例都使用这个库：

```go
import "github.com/jwwsjlm/req/v3"
```

注意不要写成 `github.com/imroc/req/v3`。这个仓库是 `jwwsjlm/req`，Go import 也必须对应这个路径。

## 先记住三句话

- `req.C()` 创建一个 HTTP client，建议长期复用。
- `client.R()` 创建一次请求，请求上的 Header、Query、Body 都放在这里。
- 先检查 `err`，再检查 `resp.IsSuccessState()`，不要只看有没有报错。

HTTP 状态码 `404`、`500` 这类服务端响应，通常不是 Go 的网络错误，所以 `err` 可能是 `nil`。业务上成不成功，要看状态码或响应内容。

## 0. 准备环境

先确认电脑里有 Go：

```sh
go version
```

这个库要求 Go `1.24+`。如果版本太低，先升级 Go。

## 1. 创建最小项目

新建一个空目录，然后初始化 Go module：

```sh
mkdir req-demo
cd req-demo
go mod init req-demo
go get github.com/jwwsjlm/req/v3@latest
```

想锁定当前文档对应版本，可以这样：

```sh
go get github.com/jwwsjlm/req/v3@v3.58.2
```

新建 `main.go`：

```go
package main

import (
	"fmt"
	"log"

	"github.com/jwwsjlm/req/v3"
)

func main() {
	client := req.C()

	resp, err := client.R().Get("https://httpbin.org/get")
	if err != nil {
		log.Fatal(err)
	}
	if !resp.IsSuccessState() {
		log.Fatalf("请求失败: %s\n%s", resp.GetStatus(), resp.String())
	}

	fmt.Println(resp.String())
}
```

运行：

```sh
go run .
```

能看到一段 JSON，就说明项目、依赖、网络请求都通了。

## 2. 推荐写法

短脚本可以直接这样：

```go
resp, err := req.Get("https://httpbin.org/ip")
```

但只要你的程序不是一次性小脚本，更推荐自己创建并复用 `client`：

```go
client := req.C()

resp, err := client.R().Get("https://httpbin.org/ip")
```

原因很简单：复用 client 才能复用连接池、Cookie、公共 Header、重试策略、代理、超时和浏览器伪装配置。

## 3. 一个稳定的基础模板

大多数 API 请求可以先从这个模板开始：

```go
package main

import (
	"fmt"
	"log"
	"time"

	"github.com/jwwsjlm/req/v3"
)

func main() {
	client := req.C().
		SetTimeout(30 * time.Second).
		SetCommonHeader("Accept", "application/json").
		SetCommonHeader("Accept-Language", "zh-CN,zh;q=0.9").
		SetCommonRetryCount(2).
		SetCommonRetryBackoffInterval(300*time.Millisecond, 3*time.Second)

	resp, err := client.R().Get("https://httpbin.org/get")
	if err != nil {
		log.Fatal(err)
	}
	if !resp.IsSuccessState() {
		log.Fatalf("请求失败: %s\n%s", resp.GetStatus(), resp.String())
	}

	fmt.Println(resp.String())
}
```

这里做了几件事：

- `SetTimeout`：给请求设置最大耗时。
- `SetCommonHeader`：给这个 client 之后发出的所有请求都加 Header。
- `SetCommonRetryCount`：网络抖动或满足重试条件时自动重试。
- `SetCommonRetryBackoffInterval`：重试间隔逐步变长，避免请求太密。

## 4. GET 请求带 Query 参数

比如请求 `https://httpbin.org/get?q=req&page=1`：

```go
package main

import (
	"fmt"
	"log"

	"github.com/jwwsjlm/req/v3"
)

func main() {
	client := req.C()

	resp, err := client.R().
		SetQueryParam("q", "req").
		SetQueryParamAny("page", 1).
		Get("https://httpbin.org/get")
	if err != nil {
		log.Fatal(err)
	}
	if !resp.IsSuccessState() {
		log.Fatalf("请求失败: %s\n%s", resp.GetStatus(), resp.String())
	}

	fmt.Println(resp.String())
}
```

多个参数也可以一次性传：

```go
resp, err := client.R().
	SetQueryParams(map[string]string{
		"q":    "req",
		"sort": "created",
	}).
	SetQueryParamAny("page", 1).
	Get("https://httpbin.org/get")
```

同名参数，比如 `tag=go&tag=http`：

```go
resp, err := client.R().
	AddQueryParams("tag", "go", "http").
	Get("https://httpbin.org/get")
```

## 5. Header、User-Agent、语言

公共 Header 放在 client 上，单次请求专用 Header 放在 `R()` 后面：

```go
package main

import (
	"fmt"
	"log"

	"github.com/jwwsjlm/req/v3"
)

func main() {
	client := req.C().
		SetUserAgent("req-demo/1.0").
		SetCommonHeader("Accept", "application/json").
		SetCommonHeader("Accept-Language", "zh-CN,zh;q=0.9")

	resp, err := client.R().
		SetHeader("X-Request-ID", "demo-001").
		Get("https://httpbin.org/headers")
	if err != nil {
		log.Fatal(err)
	}
	if !resp.IsSuccessState() {
		log.Fatalf("请求失败: %s\n%s", resp.GetStatus(), resp.String())
	}

	fmt.Println(resp.String())
}
```

Header 值不是字符串时：

```go
resp, err := client.R().
	SetHeaderAny("X-Page", 1).
	Get("https://httpbin.org/headers")
```

一个 Header 有多个值：

```go
resp, err := client.R().
	SetHeaderValues("X-Tag", "go", "http").
	Get("https://httpbin.org/headers")
```

## 6. BaseURL 和公共配置

写 API client 时，不要每次都拼完整 URL。推荐把公共域名放到 client 上：

```go
package main

import (
	"fmt"
	"log"
	"time"

	"github.com/jwwsjlm/req/v3"
)

func main() {
	client := req.C().
		SetBaseURL("https://httpbin.org").
		SetTimeout(20 * time.Second).
		SetCommonHeader("Accept", "application/json")

	resp, err := client.R().
		SetQueryParam("name", "req").
		Get("/get")
	if err != nil {
		log.Fatal(err)
	}
	if !resp.IsSuccessState() {
		log.Fatalf("请求失败: %s\n%s", resp.GetStatus(), resp.String())
	}

	fmt.Println(resp.String())
}
```

`SetBaseURL("https://httpbin.org")` 之后，请求里写 `/get` 就行。

## 7. Path 参数

普通路径参数会自动做转义：

```go
package main

import (
	"fmt"
	"log"

	"github.com/jwwsjlm/req/v3"
)

func main() {
	client := req.C().
		SetBaseURL("https://httpbin.org")

	resp, err := client.R().
		SetPathParam("id", "42").
		Get("/anything/users/{id}")
	if err != nil {
		log.Fatal(err)
	}
	if !resp.IsSuccessState() {
		log.Fatalf("请求失败: %s\n%s", resp.GetStatus(), resp.String())
	}

	fmt.Println(resp.String())
}
```

如果路径参数本身包含 `/`，并且你希望它真的作为路径的一部分，而不是被转义成 `%2F`，用 `SetPathRawParam`：

```go
resp, err := client.R().
	SetPathRawParam("path", "groups/developers/readme.md").
	Get("/anything/files/{path}")
```

## 8. POST JSON

最推荐用 `SetBodyJsonMarshal`，它会把结构体或 map 转成 JSON，并自动设置 `Content-Type: application/json; charset=utf-8`。

```go
package main

import (
	"fmt"
	"log"

	"github.com/jwwsjlm/req/v3"
)

type CreateUserRequest struct {
	Name string `json:"name"`
	Age  int    `json:"age"`
}

func main() {
	client := req.C()

	body := CreateUserRequest{
		Name: "zhangsan",
		Age:  18,
	}

	resp, err := client.R().
		SetBodyJsonMarshal(body).
		Post("https://httpbin.org/post")
	if err != nil {
		log.Fatal(err)
	}
	if !resp.IsSuccessState() {
		log.Fatalf("请求失败: %s\n%s", resp.GetStatus(), resp.String())
	}

	fmt.Println(resp.String())
}
```

如果你已经有 JSON 字符串：

```go
resp, err := client.R().
	SetBodyJsonString(`{"name":"zhangsan","age":18}`).
	Post("https://httpbin.org/post")
```

## 9. 读取 JSON 响应到 struct

如果响应是 JSON，不要总是手动 `json.Unmarshal`。可以用 `SetSuccessResult`：

```go
package main

import (
	"fmt"
	"log"

	"github.com/jwwsjlm/req/v3"
)

type SlideshowResponse struct {
	Slideshow struct {
		Title  string `json:"title"`
		Author string `json:"author"`
		Slides []struct {
			Title string `json:"title"`
			Type  string `json:"type"`
		} `json:"slides"`
	} `json:"slideshow"`
}

func main() {
	client := req.C()

	result := &SlideshowResponse{}
	resp, err := client.R().
		SetSuccessResult(result).
		Get("https://httpbin.org/json")
	if err != nil {
		log.Fatal(err)
	}
	if !resp.IsSuccessState() {
		log.Fatalf("请求失败: %s\n%s", resp.GetStatus(), resp.String())
	}

	fmt.Println(result.Slideshow.Title)
	fmt.Println(result.Slideshow.Author)
}
```

也可以先拿到响应，再调用 `Into`：

```go
result := &SlideshowResponse{}
resp, err := client.R().Get("https://httpbin.org/json")
if err != nil {
	log.Fatal(err)
}
if err := resp.Into(result); err != nil {
	log.Fatal(err)
}
```

## 10. 错误状态码怎么处理

记住：HTTP `404`、`500` 通常不会让 `err != nil`。`err` 主要代表网络错误、DNS 错误、超时、TLS 错误、请求构造错误等。

```go
package main

import (
	"fmt"
	"log"

	"github.com/jwwsjlm/req/v3"
)

func main() {
	client := req.C()

	resp, err := client.R().Get("https://httpbin.org/status/404")
	if err != nil {
		log.Fatal(err)
	}

	if resp.IsSuccessState() {
		fmt.Println("成功:", resp.String())
		return
	}

	fmt.Println("服务端返回失败状态:", resp.GetStatus())
	fmt.Println("响应体:", resp.String())
}
```

如果你的业务 API 错误响应也是 JSON，可以这样写：

```go
type APIError struct {
	Code    string `json:"code"`
	Message string `json:"message"`
}

apiErr := &APIError{}
resp, err := client.R().
	SetErrorResult(apiErr).
	Get("https://api.example.com/users/not-exist")
if err != nil {
	log.Fatal(err)
}
if resp.IsErrorState() {
	log.Fatalf("业务错误: %s %s", apiErr.Code, apiErr.Message)
}
```

上面这个地址只是示意，换成你自己的业务 API。

## 11. 表单提交

普通表单是 `application/x-www-form-urlencoded`：

```go
package main

import (

	"log"

	"github.com/jwwsjlm/req/v3"
)

func main() {
	client := req.C()

	resp, err := client.R().
		SetFormData(map[string]string{
			"username": "demo",
			"password": "secret",
		}).
		Post("https://httpbin.org/post")
	if err != nil {
		log.Fatal(err)
	}
	if !resp.IsSuccessState() {
		log.Fatalf("请求失败: %s\n%s", resp.GetStatus(), resp.String())
	}

	fmt.Println(resp.String())
}
```

表单值不是字符串时：

```go
resp, err := client.R().
```

```go
resp, err := client.R().
```

```go
package main

import (

func main() {
```

## 14. Cookie 自动保持登录态

`req.C()` 默认启用内存 CookieJar。只要复用同一个 client，服务端返回的 `Set-Cookie` 会自动保存，后续同域名请求会自动带上。

```go
package main

import (
	"fmt"
	"log"

	"github.com/jwwsjlm/req/v3"
)

func main() {
	client := req.C()

	_, err := client.R().Get("https://httpbin.org/cookies/set/session/demo123")
	if err != nil {
		log.Fatal(err)
	}

	resp, err := client.R().Get("https://httpbin.org/cookies")
	if err != nil {
		log.Fatal(err)
	}
	if !resp.IsSuccessState() {
		log.Fatalf("请求失败: %s\n%s", resp.GetStatus(), resp.String())
	}

	fmt.Println(resp.String())
}
```

手动给单次请求设置 Cookie：

```go
package main

import (
	"fmt"
	"log"
	"net/http"

	"github.com/jwwsjlm/req/v3"
)

func main() {
	client := req.C()

	resp, err := client.R().
		SetCookies(&http.Cookie{Name: "sid", Value: "abc123"}).
		Get("https://httpbin.org/cookies")
	if err != nil {
		log.Fatal(err)
	}
	if !resp.IsSuccessState() {
		log.Fatalf("请求失败: %s\n%s", resp.GetStatus(), resp.String())
	}

	fmt.Println(resp.String())
}
```

读取当前保存的 Cookie：

```go
cookies, err := client.GetCookies("https://httpbin.org")
if err != nil {
	log.Fatal(err)
}
for _, cookie := range cookies {
	fmt.Println(cookie.Name, cookie.Value)
}
```

清空 Cookie：

```go
client.ClearCookies()
```

关闭自动 Cookie：

```go
client := req.C().
	SetCookieJar(nil)
```

## 15. 多账号 Cookie 隔离

如果你要同时跑多个账号，建议每个账号一个 client：

```go
package main

import (
	"fmt"
	"log"
	"net/http"
	"net/http/cookiejar"

	"github.com/jwwsjlm/req/v3"
)

func newAccountClient() *req.Client {
	return req.C().
		SetCookieJarFactory(func() http.CookieJar {
			jar, _ := cookiejar.New(nil)
			return jar
		}).
		ImpersonateChromeWithOS(req.BrowserOSWindows)
}

func main() {
	accountA := newAccountClient()
	accountB := newAccountClient()

	_, err := accountA.R().Get("https://httpbin.org/cookies/set/account/A")
	if err != nil {
		log.Fatal(err)
	}
	_, err = accountB.R().Get("https://httpbin.org/cookies/set/account/B")
	if err != nil {
		log.Fatal(err)
	}

	respA, err := accountA.R().Get("https://httpbin.org/cookies")
	if err != nil {
		log.Fatal(err)
	}
	respB, err := accountB.R().Get("https://httpbin.org/cookies")
	if err != nil {
		log.Fatal(err)
	}

	fmt.Println("accountA:", respA.String())
	fmt.Println("accountB:", respB.String())
}
```

`SetCookieJarFactory` 的好处是：`Clone()` 出来的 client 会重新创建 CookieJar，适合账号隔离。

## 16. 超时和 Context

给 client 设置总超时：

```go
client := req.C().
	SetTimeout(10 * time.Second)
```

给单次请求设置 Context：

```go
package main

import (
	"context"
	"fmt"
	"log"
	"time"

	"github.com/jwwsjlm/req/v3"
)

func main() {
	client := req.C().
		SetTimeout(30 * time.Second)

	ctx, cancel := context.WithTimeout(context.Background(), 2*time.Second)
	defer cancel()

	resp, err := client.R().
		SetContext(ctx).
		Get("https://httpbin.org/delay/5")
	if err != nil {
		log.Fatal("请求被取消或超时: ", err)
	}
	if !resp.IsSuccessState() {
		log.Fatalf("请求失败: %s\n%s", resp.GetStatus(), resp.String())
	}

	fmt.Println(resp.String())
}
```

这个例子会请求一个延迟 5 秒的接口，但 Context 只给 2 秒，所以你会看到超时错误。

## 17. 重试

网络错误、`429`、`5xx` 常常适合重试。不要对所有请求无脑重试，尤其是会创建订单、扣款、发消息的 POST 请求，要确认业务支持幂等。

```go
package main

import (
	"fmt"
	"log"
	"time"

	"github.com/jwwsjlm/req/v3"
)

func main() {
	client := req.C().
		SetCommonRetryCount(2).
		SetCommonRetryBackoffInterval(300*time.Millisecond, 3*time.Second).
		AddCommonRetryCondition(func(resp *req.Response, err error) bool {
			if err != nil {
				return true
			}
			if resp == nil {
				return false
			}
			code := resp.GetStatusCode()
			return code == 429 || code >= 500
		}).
		AddCommonRetryHook(func(resp *req.Response, err error) {
			attempt := 0
			if resp != nil && resp.Request != nil {
				attempt = resp.Request.RetryAttempt
			}
			log.Printf("准备第 %d 次重试, err=%v", attempt, err)
		})

	resp, err := client.R().Get("https://httpbin.org/status/500")
	if err != nil {
		log.Fatal(err)
	}

	fmt.Println("最终状态:", resp.GetStatus())
}
```

## 18. 代理

本机有 HTTP 代理时：

```go
package main

import (
	"fmt"
	"log"

	"github.com/jwwsjlm/req/v3"
)

func main() {
	client := req.C().
		SetProxyURL("http://127.0.0.1:7890")

	resp, err := client.R().Get("https://httpbin.org/ip")
	if err != nil {
		log.Fatal(err)
	}
	if !resp.IsSuccessState() {
		log.Fatalf("请求失败: %s\n%s", resp.GetStatus(), resp.String())
	}

	fmt.Println(resp.String())
}
```

没有代理服务时，这段代码会连接失败。把地址换成你自己的代理地址。

需要按请求动态选择代理时，用 `SetProxy`：

```go
client := req.C().
	SetProxy(func(r *http.Request) (*url.URL, error) {
		if r.URL.Host == "example.com" {
			return url.Parse("http://127.0.0.1:7890")
		}
		return nil, nil
	})
```

这个片段需要额外 import：

```go
import (
	"net/http"
	"net/url"
)
```

## 19. 重定向

默认最多跟随 10 次重定向。禁止自动重定向：

```go
package main

import (
	"fmt"
	"log"

	"github.com/jwwsjlm/req/v3"
)

func main() {
	client := req.C().
		SetRedirectPolicy(req.NoRedirectPolicy())

	resp, err := client.R().Get("https://httpbin.org/redirect/1")
	if err != nil {
		log.Fatal(err)
	}

	fmt.Println("状态:", resp.GetStatus())
	fmt.Println("Location:", resp.GetHeader("Location"))
}
```

最多只允许 3 次重定向：

```go
client := req.C().
	SetRedirectPolicy(req.MaxRedirectPolicy(3))
```

只允许同域名跳转：

```go
client := req.C().
	SetRedirectPolicy(req.SameDomainRedirectPolicy())
```

## 20. 调试请求

开发时可以开 `DevMode()`，它会开启 dump、debug log 和 trace。生产环境不要默认全开，容易打印敏感信息，也会增加开销。

```go
package main
```

## 21. Middleware

Middleware 可以统一加 Header、记录日志、处理响应。

```go
package main

import (
	"fmt"
	"log"
	"strconv"
	"time"

	"github.com/jwwsjlm/req/v3"
)

func main() {
	client := req.C()

	client.OnBeforeRequest(func(c *req.Client, r *req.Request) error {
		r.SetHeader("X-Request-ID", strconv.FormatInt(time.Now().UnixNano(), 10))
		return nil
	})

	client.OnAfterResponse(func(c *req.Client, resp *req.Response) error {
		log.Printf("%s %s -> %s", resp.Request.Method, resp.Request.RawURL, resp.GetStatus())
		return nil
	})

	resp, err := client.R().Get("https://httpbin.org/get")
	if err != nil {
		log.Fatal(err)
	}
	if !resp.IsSuccessState() {
		log.Fatalf("请求失败: %s\n%s", resp.GetStatus(), resp.String())
	}

	fmt.Println(resp.String())
}
```

## 23. HTTP/3 推荐写法

多数场景不要一上来就强制 HTTP/3。推荐先启用 HTTP/3，并允许失败回退：

```go
package main

import (
	"fmt"
	"log"
	"time"

	"github.com/jwwsjlm/req/v3"
)

func main() {
	client := req.C().
		ImpersonateChromeWithOS(req.BrowserOSWindows).
		EnableHTTP3().
		EnableHTTP3FallbackOnError().
		SetHTTP3AltSvcFailureCooldown(30 * time.Second)

	resp, err := client.R().Get("https://tls.peet.ws/api/all")
	if err != nil {
		log.Fatal(err)
	}
	if !resp.IsSuccessState() {
		log.Fatalf("请求失败: %s\n%s", resp.GetStatus(), resp.String())
	}

	fmt.Println(resp.String())
}
```

如果你明确要强制 HTTP/3：

```go
client := req.C().
	ImpersonateChromeWithOS(req.BrowserOSWindows).
	EnableHTTP3FallbackOnError().
	EnableForceHTTP3()
```

HTTP/3 依赖 UDP、QUIC、服务端 Alt-Svc、网络环境。失败时不一定是代码问题，所以爬站或 API 调用更建议带 fallback。

## 30. 大响应不要读进内存

默认情况下，req 会自动读取响应 body，方便你用 `resp.String()`。如果响应非常大，比如大文件、流式接口，可以关闭自动读取：

```go
package main

import (
	"io"
	"log"
	"os"

	"github.com/jwwsjlm/req/v3"
)

func main() {
	resp, err := req.C().R().
		DisableAutoReadResponse().
		Get("https://httpbin.org/stream-bytes/1024")
	if err != nil {
		log.Fatal(err)
	}
	defer resp.Body.Close()

	if _, err := io.Copy(os.Stdout, resp.Body); err != nil {
		log.Fatal(err)
	}
}
```

关了自动读取后，就不要再依赖 `resp.String()`，自己从 `resp.Body` 读。

## 32. 完整业务封装：注册、登录、取数据

前面的例子都是单个请求。真实项目里，更推荐封装一个自己的业务 client，比如 `APIClient`，把注册、登录、取用户资料都写成方法。这样业务代码里就不用到处散落 `req.C()`、Header、Token、错误处理。

下面是一个完整结构示例。里面的 `https://api.demo.example` 和接口路径都是编造的，重点是看懂整体写法：怎么封装 client、怎么注册、怎么登录、怎么保存 token、怎么取数据。你接到真实项目时，把 `baseURL`、接口路径、请求字段、响应字段换成自己的就行。

```go
package main

import (
	"context"
	"errors"
	"fmt"
	"log"
	"time"

	"github.com/jwwsjlm/req/v3"
)

// APIClient 是你自己项目里的业务客户端。
// 外面调用 Register、Login、Me、ListArticles，不需要关心底层怎么拼 HTTP 请求。
type APIClient struct {
	// client 是真正发请求的 req client，公共配置都放在它上面。
	client *req.Client
	// token 是登录成功后拿到的访问令牌，后续请求会自动带上。
	token string
}

// NewAPIClient 统一创建业务客户端。
// 真实项目里把 baseURL 换成你的 API 地址，比如 https://api.example.com。
func NewAPIClient(baseURL string) *APIClient {
	return &APIClient{
		client: req.C().
			// 所有相对路径都会拼到这个 BaseURL 后面。
			SetBaseURL(baseURL).
			// 给整个 client 设置超时，避免请求一直卡住。
			SetTimeout(10 * time.Second).
			// 后续请求默认都希望服务端返回 JSON。
			SetCommonHeader("Accept", "application/json"),
	}
}

// RegisterRequest 是注册接口的请求体。
// json tag 决定最终发出去的 JSON 字段名。
type RegisterRequest struct {
	Username string `json:"username"`
	Password string `json:"password"`
	Nickname string `json:"nickname"`
}

// RegisterResponse 是注册成功后服务端返回的数据。
type RegisterResponse struct {
	ID       int    `json:"id"`
	Username string `json:"username"`
	Nickname string `json:"nickname"`
}

// LoginRequest 是登录接口的请求体。
type LoginRequest struct {
	Username string `json:"username"`
	Password string `json:"password"`
}

// LoginResponse 是登录成功后的响应。
// 这里假设服务端会返回一个 Bearer Token。
type LoginResponse struct {
	Token string `json:"token"`
}

// Profile 是登录后获取到的用户资料。
type Profile struct {
	ID       int    `json:"id"`
	Username string `json:"username"`
	Nickname string `json:"nickname"`
	Email    string `json:"email"`
}

// Article 是一个普通业务数据。
// 这里假设登录后可以拉取文章列表，你也可以换成订单、消息、商品等。
type Article struct {
	ID        int    `json:"id"`
	Title     string `json:"title"`
	CreatedAt string `json:"created_at"`
}

type ListArticlesResponse struct {
	Items []Article `json:"items"`
	Total int       `json:"total"`
}

// APIError 是服务端返回错误时的统一格式。
// 真实项目里可以加 Code、RequestID、Detail 等字段。
type APIError struct {
	Code    string `json:"code"`
	Message string `json:"message"`
}

// Register 调用注册接口。
// 方法里隐藏 HTTP 细节，调用方只需要传 RegisterRequest。
func (api *APIClient) Register(ctx context.Context, in RegisterRequest) (*RegisterResponse, error) {
	// 2xx 成功响应会自动解析到 result。
	result := &RegisterResponse{}
	// 4xx/5xx 错误响应会自动解析到 apiErr。
	apiErr := &APIError{}

	// 这条链从左到右：
	// 创建请求 -> 绑定 ctx -> 写 JSON body -> 指定成功/失败解析对象 -> POST。
	resp, err := api.client.R().
		SetContext(ctx).
		SetBodyJsonMarshal(in).
		SetSuccessResult(result).
		SetErrorResult(apiErr).
		Post("/v1/auth/register")
	if err != nil {
		return nil, err
	}
	if !resp.IsSuccessState() {
		return nil, api.responseError("register", resp, apiErr)
	}

	return result, nil
}

// Login 调用登录接口。
// 登录成功后，把 token 保存起来，并写入 client 的公共 Authorization Header。
func (api *APIClient) Login(ctx context.Context, in LoginRequest) error {
	result := &LoginResponse{}
	apiErr := &APIError{}

	// SetBodyJsonMarshal 会自动把 in 转成 JSON，并设置 Content-Type。
	resp, err := api.client.R().
		SetContext(ctx).
		SetBodyJsonMarshal(in).
		SetSuccessResult(result).
		SetErrorResult(apiErr).
		Post("/v1/auth/login")
	if err != nil {
		return err
	}
	if !resp.IsSuccessState() {
		return api.responseError("login", resp, apiErr)
	}
	if result.Token == "" {
		return errors.New("login failed: empty token")
	}

	// 保存 token，方便本地判断是否已经登录。
	api.token = result.Token
	// 关键点：后续这个 client 发出的所有请求都会自动带上：
	// Authorization: Bearer <token>
	api.client.SetCommonAuthToken(result.Token)
	return nil
}

// Me 调用需要登录的接口，获取当前用户资料。
// 因为 Login 里已经 SetCommonAuthToken，这里不需要再手动设置 Authorization。
func (api *APIClient) Me(ctx context.Context) (*Profile, error) {
	if api.token == "" {
		return nil, errors.New("not logged in")
	}

	result := &Profile{}
	apiErr := &APIError{}

	// 这里只写业务路径 /v1/me，BaseURL 已经在 NewAPIClient 里设置了。
	resp, err := api.client.R().
		SetContext(ctx).
		SetSuccessResult(result).
		SetErrorResult(apiErr).
		Get("/v1/me")
	if err != nil {
		return nil, err
	}
	if !resp.IsSuccessState() {
		return nil, api.responseError("me", resp, apiErr)
	}

	return result, nil
}

// ListArticles 演示登录后取一页业务数据。
func (api *APIClient) ListArticles(ctx context.Context, page int) (*ListArticlesResponse, error) {
	if api.token == "" {
		return nil, errors.New("not logged in")
	}

	result := &ListArticlesResponse{}
	apiErr := &APIError{}

	resp, err := api.client.R().
		SetContext(ctx).
		// Query 参数会拼成 ?page=1&page_size=20。
		SetQueryParamAny("page", page).
		SetQueryParamAny("page_size", 20).
		SetSuccessResult(result).
		SetErrorResult(apiErr).
		Get("/v1/articles")
	if err != nil {
		return nil, err
	}
	if !resp.IsSuccessState() {
		return nil, api.responseError("list articles", resp, apiErr)
	}

	return result, nil
}

// responseError 把 HTTP 状态码和服务端错误体合成一个 Go error。
// 这样业务层只需要处理 error，不需要重复拼错误信息。
func (api *APIClient) responseError(action string, resp *req.Response, apiErr *APIError) error {
	message := apiErr.Message
	if message == "" {
		message = resp.String()
	}
	if apiErr.Code != "" {
		return fmt.Errorf("%s failed: %s: %s (%s)", action, resp.GetStatus(), message, apiErr.Code)
	}
	return fmt.Errorf("%s failed: %s: %s", action, resp.GetStatus(), message)
}

func main() {
	// 这是编造的示例地址。真实项目里换成你的后端地址。
	api := NewAPIClient("https://api.demo.example")

	// 整个业务流程给 20 秒超时。
	ctx, cancel := context.WithTimeout(context.Background(), 20*time.Second)
	defer cancel()

	// 第一步：注册账号。
	user, err := api.Register(ctx, RegisterRequest{
		Username: "zhangsan",
		Password: "secret",
		Nickname: "张三",
	})
	if err != nil {
		log.Fatal(err)
	}
	fmt.Printf("注册成功: id=%d username=%s nickname=%s\n", user.ID, user.Username, user.Nickname)

	// 第二步：登录账号。登录成功后 token 会保存到 APIClient。
	err = api.Login(ctx, LoginRequest{
		Username: "zhangsan",
		Password: "secret",
	})
	if err != nil {
		log.Fatal(err)
	}
	fmt.Println("登录成功: token 已保存到 APIClient")

	// 第三步：取当前用户资料。这里会自动带上登录 token。
	profile, err := api.Me(ctx)
	if err != nil {
		log.Fatal(err)
	}
	fmt.Printf("用户资料: id=%d username=%s nickname=%s email=%s\n", profile.ID, profile.Username, profile.Nickname, profile.Email)

	// 第四步：取一页业务数据。
	articles, err := api.ListArticles(ctx, 1)
	if err != nil {
		log.Fatal(err)
	}
	fmt.Printf("文章总数: %d，本页数量: %d\n", articles.Total, len(articles.Items))
}
```

真实项目里你只需要保留这几类代码：

- `APIClient`：保存 `*req.Client`、token、公共配置。
- `NewAPIClient`：统一设置 `BaseURL`、超时、公共 Header。
- `Register`、`Login`、`Me`、`ListArticles`：每个业务接口一个方法。
- `APIError`：统一解析服务端错误响应。

登录成功后这两行最关键：

```go
api.token = result.Token
api.client.SetCommonAuthToken(result.Token)
```

后续 `api.Me(ctx)` 和 `api.ListArticles(ctx, 1)` 不需要再手动设置 Header，因为 `SetCommonAuthToken` 已经给这个 client 后续请求统一加了：

```txt
Authorization: Bearer <token>
```

如果你的后端是 Cookie 登录，而不是 Bearer Token，就不要 `SetCommonAuthToken`。保持复用同一个 `api.client`，req 默认 CookieJar 会自动保存登录接口返回的 `Set-Cookie`。

## 33. 常见问题

`go get` 报错找不到包：

确认 import 和命令都是 `github.com/jwwsjlm/req/v3`，并且已经执行过 `go mod init`。

```sh
go mod init your-project-name
go get github.com/jwwsjlm/req/v3@latest
go mod tidy
```

`resp.String()` 是空的：

常见原因有三个。第一，你用了 `SetOutputFile`，响应体已经写到文件。第二，你用了 `DisableAutoReadResponse`，需要自己读 `resp.Body`。第三，服务端本来就没有返回 body，比如很多 `204`、`HEAD`、`/status/404` 接口。

Cookie 没有保持住：

确认你复用了同一个 `client := req.C()`。如果每次请求都重新 `req.C()`，CookieJar 也是新的，自然没有上一次的 Cookie。

HTTP/3 没成功：

HTTP/3 要看服务端、UDP、网络环境、Alt-Svc、代理支持情况。普通使用建议 `EnableHTTP3().EnableHTTP3FallbackOnError()`，不要一开始就强制。

浏览器伪装后还是被拦：

HTTP 指纹只是风控的一部分。站点还可能看 IP、账号、Cookie 历史、访问频率、行为路径、JS 环境和验证码。先用 `https://tls.peet.ws/api/all` 看 HTTP 层指纹，再排查其它因素。

什么时候用 `DevMode()`：

本地调试用。生产环境不要默认开启，因为它可能打印 Header、Cookie、Token、请求体和响应体。
